# Caker 用户指南

面向**在浏览器里使用 Caker** 的人。项目介绍见 [README](../README.md)；开发验收见 [里程碑进度](milestones.md)、[文档索引](README.md)。

## 1. 启动与访问

```bash
cd caker
cp .env.example .env
# 编辑 .env：LLM_*、WORKSPACE_ROOT 等
pip install -e .
uv run uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

浏览器打开：**http://127.0.0.1:8000/**

顶栏绿点表示 `/health` 正常；灰点表示后端未就绪。

## 2. 界面概览

```
┌─────────────┬──────────────────────────────┐
│ 侧栏        │ 顶栏：标题 · 流式 · 状态点    │
│ · 新对话    ├──────────────────────────────┤
│ · 历史列表  │ 消息区                        │
│ · 工作区    ├──────────────────────────────┤
│ · 用户管理  │ 输入框 · + 上传 · 发送        │
└─────────────┴──────────────────────────────┘
```

- **新对话**：创建新 `session_id`，对应新的会话工作区目录。
- **流式**：开启时逐字输出；关闭时走非流式图（可走 `result_set` 交卷）。
- **User ID**：左下用户列表切换；影响工作区路径与 MemPalace 记忆范围（请求头 `x-user-id`）。

## 3. 工作区

每个「用户 + 会话」在服务器上有独立目录：

```text
var/workspace/<user_id>/<session_id>/
  data/uploads/    ← Web 上传的文件
  outputs/         ← Agent 写入结果
  skills/          → 仓库 skills/ 的符号链接（只读）
```

侧栏 **工作区** 面板：

| 区域 | 说明 |
|------|------|
| 用户 / 会话 | 当前身份，不重复显示完整磁盘路径 |
| 工作目录 | 相对路径（如 `local/chat-xxx`）；悬停可看 WSL 与 Windows 路径 |
| 文件列表 | 已上传或目录内文件摘要 |
| 复制路径 | Win+WSL 优先复制 `\\wsl.localhost\...`，便于粘贴到资源管理器 |
| 打开资源管理器 | 在**运行 uvicorn 的本机**打开该文件夹（不是远程浏览器所在电脑） |

### Win + WSL

服务跑在 WSL 内时，会自动把 Linux 路径转为 `\\wsl.localhost\<发行版>\...` 并用 `explorer.exe` 打开。

### Mac / Linux 桌面

分别使用 `open` / `xdg-open` 打开 Finder 或系统文件管理器。

### 纯服务器 / 无图形界面

按钮可能返回错误；请用「复制路径」在本地挂载或 SSH 后自行打开。

## 4. 上传与对话

1. 点击输入框左侧 **+**，选择文件（有大小上限，见服务端配置）。
2. 文件写入当前会话的 `data/uploads/<文件名>`。
3. 发送消息时，附件会随请求带给 Agent；模型可通过 **read** 等工具读取。

提示：若历史消息里写了 `[文件] data/uploads/xxx` 但文件已被删，发送前会提示重新上传。

## 5. 会话与数据存哪

| 内容 | 位置 | 说明 |
|------|------|------|
| 对话列表标题、设置 | `var/web/` | 换浏览器、清缓存可能丢列表；连同一后端可恢复服务端数据 |
| 多轮聊天内容 | `var/state.db` | 同一 `session_id` 续聊依赖检查点 |
| 上传文件、Agent 产出 | `var/workspace/...` | 删会话时应一并清理（管理 API / 删对话） |
| 跨会话语义记忆 | `var/chroma/` | 需配置 `EMBEDDING_*`；同 User ID 可召回 |

## 6. Agent 能做什么（用户视角）

- 读写当前会话工作区内的文件（read / write / glob / edit 等）。
- 调用 **skills** 目录下的技能说明与脚本。
- 长对话时自动**压缩上下文**（保留系统提示与当前轮），减少超长报错。
- 流式模式下可见工具执行、压缩等**状态提示**（非最终正文）。

具体工具列表以运行时的系统提示与 MCP 注册为准。

## 7. 常见问题

**侧栏工作区显示「失败」或 Not Found**  
重启 uvicorn 后硬刷新页面（Ctrl+Shift+R）。

**Agent 说工作区没有文件，但侧栏能看到**  
多为会话 ID 不一致；确认当前对话与上传时同一会话。若刚修过工具配置，需重启后端。

**打开资源管理器没反应**  
确认 uvicorn 跑在你本机 WSL/Windows/Mac 上；远程服务器上点按钮只会尝试在服务器主机打开。

**想在本机 VS Code 里打开工作区**  
用「复制路径」粘贴到 Cursor/VS Code 的 Open Folder，或使用 WSL 路径 `\\wsl.localhost\...`。
